Read / Write Separation

Falcon is strictly read-only intelligence: markets, candles, trades, wallet analytics, social pulse, and anomaly signals. It enriches strategy evaluation and portfolio visibility. It does not place, cancel, or manage orders.

Execution Authority

All trade writes route through the backend into Polymarket CLOB after signal approval and risk validation. Telegram does not act as a manual order terminal.

Security Protocol: intelligence layers are read-only, execution lives behind backend risk approval, and user controls are separated from trade authority.

Each user registers their own Falcon key via /setkey. The backend stores it encrypted and injects a per-user client when needed. This preserves multi-user isolation and avoids shared intelligence sessions.

Enforced Logic Rules

·
Positive EV Gate: orders rejected if EV is not positive after slippage and fee modeling.
·
Liquidity Floor: target market must meet minimum executable depth before entry approval.
·
Idempotency: duplicate intent hashing prevents repeated order submission within the same event window.
·
MDD Auto-Pause: live auto-trading is disabled automatically if session loss exceeds threshold.
