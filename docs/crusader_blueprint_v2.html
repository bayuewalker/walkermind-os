Design Principles
01
Risk-First
Every signal is filtered through hard risk rules before execution. Kelly, position limits, drawdown circuit-breakers, and kill switch are enforced in code — not just configured.
02
Non-Custodial
Users own their wallets. The backend orchestrates execution but never holds funds. Each user authenticates with their own Polygon wallet — private keys never leave their device.
03
Read / Write Separation
Falcon MCP is purely read-only intelligence. CLOB is purely write-side execution. They never cross — preventing accidental order submission from the intelligence layer.
04
Fully Automated
After setup, zero manual intervention required. Bot scans markets, runs ML-enhanced analysis, applies risk rules, and executes independently 24/7 on Fly.io.

07Strategies S1–S5All backed by Falcon read-layer
S1 — Narrative Momentum
Falcon #585 social spike + market-lag measurement → EV edge gate → ENTER/SKIP. Enters when market hasn't repriced after news breaks. Falcon is the read source for acceleration + volume signals.
S2 — Cross-Exchange Arbitrage
Falcon #565/#573 Kalshi markets/trades → normalized probability + actionable spread gate + fee/slippage net edge. CLOB is used for order submission only; Falcon reads Kalshi side.
S3 + S3.1 — Smart-Money Copy
Falcon #581 Wallet360 + #584 H-Score: consistency / discipline / frequency / diversity → confidence-weighted ENTER/SKIP. Multi-dimensional wallet quality gate derived entirely from Falcon read layer.
S4 — ML-Weighted Aggregator
Collects outputs of S1/S2/S3/S5 → ML-driven weights (P15) + regime modifiers (P11) + Falcon alpha layer (P14.3) → single trade decision. Output: selected_trade / decision.
S5 — Settlement Gap Scanner
Falcon #565/#573 Kalshi resolution detected → Polymarket equivalent market → resolved-outcome price < 0.95 + liquidity guard → ENTER via CLOB, source="settlement_gap".
P14.3 — Falcon Alpha Layer
Smart-money flow concentration + momentum (price jumps, candle trend) + liquidity (spread/depth) → bounded external_signal_weight fed to S4. All inputs from Falcon read agents. Insufficient data → neutral weight.

12Falcon API Key SetupPer-user · AES-256 · ketanalytics.com
3-Step Setup Flow
Step 1 — Get Key
User signs up at ketanalytics.com → API Keys page → copies Bearer JWT token starting with eyJ...
Step 2 — Register
User sends /setkey eyJhbG... to CrusaderBot → bot calls FalconClient.validate_key() → AES-256 encrypted, stored in user_settings.falcon_api_key
Step 3 — Isolated Use
Every Falcon call uses get_falcon(user_id, db) — decrypts that user's key, creates fresh FalconClient. Zero cross-user leakage.
falcon/client.py
# server/integrations/falcon/client.py class FalconClient: BASE = "https://narrative.agent.heisenberg.so" def __init__(self, api_key: str, timeout: float = 30.0, max_retries: int = 3): self._headers = {"Authorization": f"Bearer {api_key}"} self._timeout = timeout self._retries = max_retries # All methods are READ-ONLY — market intelligence only, no order submission async def get_markets(self, ...) -> list # #574 async def get_trades(self, ...) -> list # #556 async def get_candlesticks(self, ...) -> list # #568 async def get_orderbook(self, ...) -> list # #572 async def get_market_360(self, ...) -> list # #575 async def get_wallet_360(self, ...) -> list # #581 async def get_h_leaderboard(self, ...) -> list # #584 async def get_social_pulse(self, ...) -> list # #585 async def get_pnl(self, ...) -> list # #569 async def get_wallet_stat(self, ...) -> list # #586 async def get_price_jumps(self, ...) -> list # #596 async def get_leaderboard(self, ...) -> list # #579 async def get_kalshi_markets(self, ...) -> list # #565 async def get_kalshi_trades(self, ...) -> list # #573 # Per-request isolation — called by every backend service that needs Falcon def get_falcon(user_id: UUID, db: AsyncSession) -> FalconClient: s = UserSettingsRepo(db).get(user_id) if not s or not s.falcon_api_key: raise ServiceError("Add your Falcon API key via /setkey") raw_key = aes_decrypt(s.falcon_api_key, settings.ENCRYPTION_KEY) return FalconClient(api_key=raw_key)
❌ NOT in .env
Falcon API keys are never in environment variables. The server .env holds only ENCRYPTION_KEY used to encrypt/decrypt keys stored in DB.
✅ In DB (AES-256 encrypted)
Table: user_settings.falcon_api_key
Stored as: AES-256-CBC ciphertext
Decrypted at: runtime, per-request only
Never logged, never returned in API responses
