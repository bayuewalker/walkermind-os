## 1. OVERVIEW

Phase 10.4 runs the full PolyQuantBot trading pipeline against the **real Polymarket WebSocket feed** while keeping all execution in paper (simulation) mode. No real orders are ever sent. The goal is to measure real-market latency, slippage behavior, WS stability, and pipeline performance under live conditions.

**Pipeline:**
```
PolymarketWSClient (wss://clob.polymarket.com)
    │  live orderbook / trade events
    ▼
OrderBookManager → Phase7MarketCache
    │  microstructure (bid/ask/spread/depth)
    ▼
decision_callback (signal generator)
    │  signal_generated_ts stamped
    ▼
RiskGuard.check()        ← kill switch / daily loss / drawdown
    │
ExecutionGuard.validate() ← liquidity / slippage / size / dedup
    │
ExecutionSimulator.execute() ← PAPER_LIVE_SIM (no real orders)
    │  fill + slippage recorded in FillTracker
    ▼
MetricsValidator (ev_capture / fill_rate / p95_latency / drawdown / slippage)
    │
TelegramLive (error / kill / checkpoint alerts)
```
