# production_bootstrap — FORGE-X Report

## 1. What Was Built

Production-grade bootstrap module for PolyQuantBot that enables zero-config deployment.

Key capabilities:
- **Strict credential validation**: all required credentials are checked at startup; any missing value raises a `RuntimeError` with a clear, actionable message before the pipeline starts.
- **Optional config auto-fill**: every non-critical env var has a safe production default — the system starts correctly even if only the five required credentials are set.
- **Automatic market discovery**: when `MARKET_IDS` is not set, the Gamma REST API is queried, markets are filtered by liquidity (>10k USD), and the top-N by volume are selected dynamically.
- **Hard fail on empty discovery**: if the Gamma API returns zero qualifying markets, the system raises `RuntimeError` (correct behavior — never start with no markets).
- **Startup log**: structured log entry emitted with mode, market count, and selected IDs.

## 2. Current System Architecture

```
Bootstrap sequence (core/bootstrap.py)
  │
  ├─ validate_credentials()
  │     Checks: CLOB_API_KEY, CLOB_API_SECRET, CLOB_API_PASSPHRASE,
  │             TELEGRAM_TOKEN (or TELEGRAM_BOT_TOKEN), TELEGRAM_CHAT_ID
  │     → RuntimeError on any missing value
  │
  ├─ build_config()
  │     Reads all optional env vars; fills production defaults
  │     → pipeline config dict (websocket, go_live, risk, metrics, execution_guard)
  │
  ├─ discover_markets(cfg)
  │     If MARKET_IDS set → use directly
  │     Else → GET gamma-api.polymarket.com/markets
  │              filter active + volume ≥ MIN_LIQUIDITY_USD
  │              sort desc by volume
  │              take top MAX_MARKETS (default 5)
  │     → RuntimeError if result is empty
  │
  └─ log_startup(mode, market_ids)
        Structured log: mode, markets_selected, market_ids

main.py
  │
  └─ run_bootstrap() → (cfg, market_ids)
       ├─ trading_mode = cfg["go_live"]["mode"]
       ├─ LIVE guard: ENABLE_LIVE_TRADING must be "true"
       └─ start LivePaperRunner (PAPER) or Phase10PipelineRunner (LIVE)
```

## 4. What's Working

- ✅ `validate_credentials()` hard-fails on any missing required credential with a descriptive message listing all missing variables
- ✅ `validate_credentials()` accepts `TELEGRAM_BOT_TOKEN` as an alias for `TELEGRAM_TOKEN`

- ✅ `discover_markets()` handles both list and dict-shaped Gamma API responses
